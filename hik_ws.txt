海康威视私有 WebSocket 通信协议解密流程分析:

1. 通路握手与认证：
网络建立 WebSocket（多数为 wss）的连接后，向服务端请求特定的 media 端点 URL。
服务端的 WebSocket 返回一条携带授权要求信息的 JSON TEXT 数据。主要包含：
 - `PKD`：由服务器下发的 RSA 公钥（十六进制编码）。
 - `rand`：服务器下发的随机质询字符串，用于产生加密 Auth 信息。
 - `cipherSuite`：支持的加密套件序号。

2. 客户端密钥对生成与加密传输：
客户端根据时间戳生成 AES 的 `iv` 和 `key` 原始明文。
随后使用海康威视固化的硬编码密钥 (HIK_FIXED_KEY: 1234567891234567123456789123456712345678912345671234567891234567 和 HIK_FIXED_IV: 12345678912345671234567891234567) 对时间戳进行 AES-128-CBC 加密，得到真正的当前会话的 `iv` 与 `key`。

在此基础上：
 - realplay_key 字段：利用服务器下发的 `PKD`（RSA 模数）与硬编码的公钥指数（65537），采用自定义填充的 RSA 算法，对 `iv:key` 的组合字符串进行加密并转换成 Hex 字符串。由于前端 JS 代码采用了修改版的 PKCS1-v1.5 填充（在前方插入多位随机非零字节并以0x00 0x02开头），此加密值能够被设备识别用来当成后续的解密 Key 和 IV。
 - authorization 字段：利用刚刚产生的 `key` 和 `iv` 对形如 `[rand]:[password]` 的字符串进行 AES-CBC 加密得到。
 - token 字段：采用同样的 `key` 和 `iv` 对 SHA-256 计算后的 realplay 设备 URL 进行 AES-CBC 加密获取。

客户端将这三个密文内容组织在一个 `realplay` 的 JSON 指令结构中发送给服务器。

3. 视频流数据的解密与接收：
当客户端与服务端的认证通过以后，服务端就会不断下发二进制类型的 WebSocket 数据帧（opcode=0x02 或 0x40/0x41 协议头）。
通常如果是经过协议约定的，服务器下发的视音频二进制流将被客户端传递过来的会话 `key` 和 会话 `iv` 进行 AES-CBC 或者相关对称加密加密过。（在此客户端中，暂未看到对二进制流内容本身的 AES 解密过程，可能服务器若配置为不支持或返回的空 `key` 则直接发送了未加密的码流报文或明文 PS 流格式。若为加密，则需依据 negotiated 的 cipherSuite 利用会话 `key` 进行对等解密拆包）。

数据流解码后，即还原出 H.264/H.265 的裸流（或者封装在 PS/TS 的流中），再交由底层的如 FFmpeg、OpenCV 或 WASM 解码器进行按帧视频还原。
